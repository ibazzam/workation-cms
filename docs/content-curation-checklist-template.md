# Content Curation Checklist Template

Use this checklist for each listing or listing bundle under review.

## A) Intake
- [ ] Request source captured (onboarding, update, escalation, audit)
- [ ] Priority assigned (`P0`/`P1`/`P2`)
- [ ] Owner assigned

## B) Metadata Completeness
- [ ] Name/title and category are valid
- [ ] Location and island/atoll metadata are complete
- [ ] Price baseline and availability windows are present
- [ ] Policy fields (cancel/refund/no-show) are complete
- [ ] Required media and summaries are present

## C) Quality and Editorial
- [ ] Taxonomy mapping is correct
- [ ] Description quality meets editorial baseline
- [ ] Duplicate/noisy content removed
- [ ] Media ordering and quality validated

## D) Trust and Safety
- [ ] No prohibited or misleading claims
- [ ] External/social references pass safety checks
- [ ] Moderation escalations resolved or tracked

## E) Publish QA
- [ ] Customer-facing preview validated
- [ ] Publish decision recorded (`APPROVED`/`NEEDS_CHANGES`/`HOLD`)
- [ ] Reviewer and timestamp captured

## F) Recertification
- [ ] Next recertification date scheduled
- [ ] Drift/complaint watch tags applied
